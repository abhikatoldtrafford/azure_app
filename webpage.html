üí¨ Chatbot API

POST /initiate-chat

Initialize a new chat session with an AI assistant. Creates an assistant, thread, and vector store. Optionally upload a file and set initial context.

Parameters

Parameter Type Required Description
context string Optional Initial context or persona for the assistant
file file Optional File to upload (PDF, DOCX, images, CSV, Excel)

üß™ Live Test

cURL Example 
curl -X POST "${window.location.origin}/initiate-chat" \
  -F "context=You are a helpful assistant specialized in data analysis" \
  -F "file=@document.pdf"
Response Example 
{
  "message": "Chat initiated successfully.",
  "assistant": "asst_abc123...",
  "session": "thread_xyz789...",
  "vector_store": "vs_def456..."
}
GET /conversation

Send a message to the assistant and receive a streaming response (Server-Sent Events). Perfect for real-time chat interfaces.

Parameters

Parameter Type Required Description
session string Required Thread ID from initiate-chat
assistant string Required Assistant ID from initiate-chat
prompt string Required User message

üß™ Live Test

cURL Example (Streaming) 
curl -N "${window.location.origin}/conversation" \
  -G --data-urlencode "session=thread_abc123" \
  --data-urlencode "assistant=asst_xyz789" \
  --data-urlencode "prompt=Analyze the sales data from the uploaded file"
POST /upload-file

Upload a file to an existing assistant session. Supports documents, images, and data files.

üß™ Live Test

cURL Example 
curl -X POST "${window.location.origin}/upload-file" \
  -F "file=@sales_data.csv" \
  -F "assistant=asst_xyz789" \
  -F "session=thread_abc123" \
  -F "prompt=Analyze this sales data"

üí° Content & Idea Generation API

POST /completion

Generate creative content, ideas, documents, or structured data using AI. Supports multiple output formats including Word documents, Excel files, and CSV.

Parameters

Parameter Type Required Description
prompt string Required Main generation prompt
output_format string Optional Format: docx, csv, excel, or none (text)
system_message string Optional Additional context or guidelines
rows_to_generate integer Optional For CSV/Excel: number of rows (default: 30)

üß™ Live Test

Generate Business Plan (DOCX) 
curl -X POST "${window.location.origin}/completion" \
  -F "prompt=Create a comprehensive business plan for a SaaS startup in the HR tech space" \
  -F "output_format=docx" \
  -F "system_message=Include market analysis, 5-year financial projections, and go-to-market strategy"

‚≠ê Reviews & Data Processing API

POST /extract-reviews

Extract reviews and structured data from documents or generate synthetic review datasets. Perfect for creating test data or analyzing customer feedback.

Parameters

Parameter Type Required Description
file file Optional File to extract data from
prompt string Optional Instructions for extraction/generation
mode string Optional auto (default), extract, or generate
columns string Optional Column names or "auto" (default)
output_format string Optional excel (default), csv, or json
rows_to_generate integer Optional For generation mode (default: 30)

üß™ Live Test

Generate Synthetic Reviews 
curl -X POST "${window.location.origin}/extract-reviews" \
  -F "mode=generate" \
  -F "prompt=Generate realistic product reviews for wireless headphones" \
  -F "columns=customer_name,rating,review_title,review_text,verified_purchase,helpful_count" \
  -F "rows_to_generate=50" \
  -F "output_format=excel"

‚ùó Error Codes & Troubleshooting

Common Error Codes

Code Meaning Solution
400 Bad Request Check your request parameters and format
404 Not Found Verify the endpoint URL and resource IDs
422 Unprocessable Entity Check data format and content requirements
500 Internal Server Error Retry the request; contact support if persists
503 Service Unavailable Service temporarily down; retry later

Best Practices

  • Always handle errors gracefully in your application
  • Implement exponential backoff for retries
  • Store assistant and session IDs for conversation continuity
  • Validate file formats before uploading
  • Use appropriate output formats for your use case
  • Monitor health endpoints for service availability
